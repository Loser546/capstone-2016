  • Project Purpose: Creating an application that collects and displays health information about individual patients, provides standard reporting and scheduling capabilities, and employs standard terminology. The software should include international classifications of medical standards such as ICD-10 coding and useful features for medical advice generation.

Project Goals and Objectives:

  1. To have a functional database that can store the patient information in a way that can be presented in a user-friendly interface.
  2. To have an application that can reliably analyze and manage the information in the database. It will be split into five modules:
    • Patient module: Direct interface to enter patient data
    • Appointment module: Direct interface to enter appointments, based on physician availability
    • Medical Advice module: Should analyze diagnosis and recommend medication/course of action
    • Reporting module: Read only interface to Medical Advice module
    • Database Management module: To manage patient database

Success Criteria:

The completion of both goals.
